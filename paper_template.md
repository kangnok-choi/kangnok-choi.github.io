# 논문 짧은 제목

##### *저자 et al.* 논문 풀 타이틀 *(연도 학회)*

## 1. 논리의 흐름 (The Logic Flow)
*저자의 주장(Storyline)을 해체하여 정리합니다.*

* **문제 정의 (Problem Definition):**
    * 기존 연구의 한계점(Pain Point)은 무엇인가?
    * 이 논문이 해결하려는 구체적인 문제는 무엇인가? (Why this paper?)
* **노벨티 (Novelty / Key Method):**
    * *지침: 문제 해결을 위한 기술적 장치(Fact)에 집중하세요.*
    * 기존 방법론(SOTA)과 구조적으로 무엇이 다른가? (예: 아키텍처 변경, 새로운 Loss 함수, 데이터셋 구축 방식 등)
* **컨트리뷰션 (Contribution / Impact):**
    * *지침: 노벨티를 통해 얻어진 효용(Benefit)에 집중하세요.*
    * 이 방법론이 학계나 산업계에 주는 실질적 이득은 무엇인가? (예: 성능 SOTA 달성, 연산 효율성 증대, 새로운 태스크 정의)

## 2. 검증과 증명 (Verification)
*실험 설계의 타당성을 평가합니다.*

* **메인 결과:** 제안하는 방법의 우수성을 입증하는 결정적 수치나 그래프.
* **검증 논리 (Ablation Study):**
    * 저자는 자신의 아이디어가 유효함을 어떻게 증명했는가?
    * 대조군(Baseline) 설정은 공정한가?

## 3. 비판적 사고 (Critical Thinking) 🔥 [가장 중요]
*리뷰어의 시각으로 논문의 약점을 파고드세요.*

* **강점 (Strengths):** 배울 만한 연구 설계나 참신한 발상.
* **한계점 (Weaknesses / Limitations):**
    * 저자가 밝힌 한계점 외에, **당신이 발견한 치명적인 약점**은 무엇인가?
    * 실험에서 의도적으로 배제된 비교 대상(Alternative)은 없는가? (예: "왜 LoRA랑 비교 안 했지?", "왜 Single-task 성능은 없지?")
    * 특정 상황(Edge case)에서 실패할 가능성은 없는가?
* **내 연구에의 적용 (Action Plan):**
    * 내 연구에 당장 적용할 수 있는 구체적인 아이디어, 코드 로직, 데이터셋 구축 팁.
